IDQN vs QMIX — Algorithm Comparison
Baseline
IDQN
Independent DQNs — naive multi-agent extension
Training Paradigm
Fully Decentralized
Each agent trains its own DQN independently, treating other agents as part of the environment. No shared information, no joint optimization.
Team Value Function
None
No centralized Q-value. Each agent optimizes its own individual return signal, which may conflict with team objectives.
Credit Assignment
Individual Only
Cannot distinguish between "my action was good" and "partner's action caused the reward." Multi-agent credit assignment is entirely unaddressed.
Non-Stationarity
Unmitigated
From any agent's perspective, the "environment" is non-stationary because the other agent is also updating. Breaks convergence guarantees of standard Q-learning.
IGM Property
Not Guaranteed
Individual greedy actions are not guaranteed to be globally optimal for the team. Coordination can fail even when individual policies look reasonable in isolation.
Computational Cost
Minimal
No mixing network, no hypernetworks. O(n) in the number of agents. Simple to implement.
Implemented
QMIX
Centralized Training, Decentralized Execution
Training Paradigm · CORE DIFFERENCE
CTDE — Centralized Training
Training uses the full joint state z = [s₀; s₁]. A centralized mixing network jointly optimizes Q_tot, conditioning mixing weights on global context through hypernetworks. Execution remains decentralized.
Team Value Function
Q_tot via Mixing Network
A shared centralized value Q_tot is computed from individual Q-values through a hypernetwork-conditioned mixing network. Directly optimized during training.
Credit Assignment
Value Decomposition
The mixing network factors Q_tot into individual contributions. The gradient of the loss flows back through the mixer into each agent's DQN — giving each agent a credit signal derived from the team outcome.
Non-Stationarity
Partially Mitigated
Joint optimization via the centralized Q_tot provides a more stable learning signal. The target networks for all three components (Q̄₀, Q̄₁, fmix_target) further reduce instability.
IGM Property
Architecturally Guaranteed
Softplus on mixing weights ensures ∂Q_tot/∂Qᵢ > 0. Monotonicity is enforced, not assumed. Decentralized greedy actions are provably consistent with the team optimum.
⚡ What QMIX adds over IDQN
Hypernetwork-conditioned mixing network (4 single-layer NNs)
Centralized state z ∈ ℝ¹⁹² as training input
Target mixing network (soft-updated)
Softplus weight constraint → strict IGM guarantee
