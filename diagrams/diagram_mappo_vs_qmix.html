Why QMIX? — Algorithm Selection for Cooperative MARL
Context: Multiple MARL algorithms exist for cooperative tasks — MAPPO, MADDPG, QMIX. The choice matters structurally: each makes different assumptions about the action space, training paradigm, and how it addresses credit assignment. For the Overcooked setting (discrete actions, fully cooperative, sparse rewards, slow simulator), these differences are decisive.
Considered — not chosen
MAPPO
Multi-Agent PPO · On-Policy Actor-Critic
Training Paradigm
On-Policy
Cannot reuse past experience. Every update step requires fresh environment interaction — expensive in a simulator as slow as Overcooked. Replay buffers are not compatible with on-policy methods.
Team Value Function
Centralized V(s) — Critic
A shared state-value function V(s) is used as the centralized critic — it estimates expected return but does not decompose into per-agent contributions. No value decomposition.
Credit Assignment
Advantage Estimation (GAE)
Relies on Generalized Advantage Estimation (GAE) to derive per-agent credit from centralized V(s). Indirect and statistically noisy — does not produce an explicit per-agent utility signal.
IGM Property
Not Guaranteed
No monotonicity constraint. Local greedy actions are not provably consistent with the globally optimal joint action. Coordination correctness is statistically encouraged, not architecturally enforced.
Hyperparameter Surface
Broader — additional knobs
PPO introduces: clip range ε (controls policy update step size), GAE parameter λ (controls advantage estimation bias-variance tradeoff), and entropy coefficient (controls exploration). Each requires tuning in addition to the standard RL hyperparameters.
Action Space Fit
Discrete — Compatible
MAPPO uses a categorical policy distribution over discrete actions — compatible with Overcooked's 6-action space. (Unlike MADDPG, which requires continuous actions.)
Chosen ✓
QMIX
Monotonic Value Decomposition · Off-Policy Q-Learning
Training Paradigm
Off-Policy + Replay Buffer
Decouples data collection from optimization. Past transitions can be reused many times — making training significantly more sample efficient in environments where each episode is expensive to run.
Team Value Function · CORE
Q_tot via Value Decomposition
Q_tot is explicitly factored into individual agent utility functions Qᵢ through a hypernetwork-conditioned mixing network. The mixing network directly learns how each agent's Q-value contributes to team value.
Credit Assignment
Explicit — via Mixing Gradients
Gradients of the TD loss flow through the mixing network back into each agent's DQN — each agent receives a credit signal derived directly from its contribution to Q_tot. Explicit and architecturally grounded.
IGM Property
Architecturally Guaranteed
Softplus activation on mixing weights ensures ∂Q_tot/∂Qᵢ > 0 always. Monotonicity is built into the network structure — greedy local actions are provably consistent with the global optimum.
Hyperparameter Surface
Leaner — no PPO-specific knobs
No clip range, no GAE parameter, no entropy coefficient. Core knobs: γ (discount), α (learning rate), τ (soft-update), λ (ε-decay). Each of these is already necessary — QMIX adds none beyond standard Q-learning.
Tradeoff — The Catch
ε-Decay Is Extremely Sensitive
Being off-policy means exploration must be managed externally through ε-greedy decay. In cooperative sparse-reward tasks, this single parameter is a razor's edge — the wrong value causes complete training failure. This sensitivity is what gave the project its name: Chaos to Cuisine.
