Centralized information: concatenation of both agents' 96-dim state vectors, giving a 192-dim input. This conditions the hypernetworks, allowing mixing weights to be state-dependent and context-aware.

Individual agent utility values for the actions taken in this transition. Gathered from Q₀ and Q₁ using the action indices as indices (.gather(1, action_idx)). Stacked into a 2-vector per batch item.
HyperNet 1a
fc_w1: W₁

→ Softplus
→ reshape (B,64,2)
Produces W₁, the weight matrix for mixing layer 1. Linear projection from z (192-dim) to 128-dim. Softplus ensures all outputs are strictly positive. Reshaped to (B, 64, 2) — a batch of (hidden_dim × num_agents) matrices.
HyperNet 1b
fc_b1: B₁
Linear(192 → 64)
no activation
shape: (B, 64)
Produces B₁, the bias vector for mixing layer 1. No Softplus — biases are unconstrained because they don't affect the partial derivative ∂Q_tot/∂Qᵢ. Shape: (B, 64) — one bias per hidden unit per batch item.
HyperNet 2a
fc_w2: W₂
Linear(192 → 64)
→ Softplus
→ reshape (B,1,64)
Produces W₂ for mixing layer 2. Softplus for positivity. Reshaped to (B, 1, 64) — a batch of (1 × hidden_dim) row vectors that collapse the hidden layer to a scalar.
HyperNet 2b
fc_b2: B₂
Linear(192 → 1)
no activation
scalar per batch
Produces B₂, a scalar bias for the output layer. State-conditioned — gives Q_tot a global offset that depends on the full joint state, improving expressivity without violating monotonicity.
Layer 1 · Mixing

Batched matrix multiply: W₁ (B,64,2) × [Q₀;Q₁] (B,2,1) → (B,64,1). Add B₁. Apply ELU activation. ELU is used (not ReLU) for smoother gradient flow. Output h is a 64-dim hidden representation per batch item.

Batched matrix multiply: W₂ (B,1,64) × h (B,64,1) → (B,1,1). Add B₂ scalar. Squeeze to (B,) — one Q_tot value per batch item. No activation on output (Q-values can be any real number).

centralized value

The global centralized action-value Q_tot. Represents the expected team return given the joint state z and joint action [a₀, a₁]. Used to compute the Bellman target and the Smooth L1 training loss.
⚡ ∂Q_tot/∂Qᵢ > 0 · Monotonicity Guaranteed
Softplus vs. |·|
Original QMIX uses hard absolute value for weight positivity. This implementation uses Softplus — differentiable everywhere, no zero-gradient region, smoother and more stable training throughout.
